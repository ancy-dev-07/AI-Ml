Git & GitHub Documentation

What is Git?

  • Git is a Version Control System
  • Tracks changes in source code
  • Allows collaboration between developers

What is GitHub?

GitHub is a web-based platform that hosts Git repositories and helps developers collaborate, review code, and manage projects.

GitHub Alternatives

  • Bitbucket
  • GitBucket

Creating a Repository on GitHub

  1. Click New Repository
  2. Enter repository name
  3. Click Create Repository
  4. Add a new file or upload files
  5. Commit changes

Installing & Configuring Git

Check Git installation:

git

Configure username and email:


git config --global user.name "ancy-dev-07"
git config --global user.email "ancys722@gmail.com"

Cloning a Repository

git clone <repository-link>
cd repository-folder

Basic Git Workflow


git status
git add .
git commit -m "message"
git push origin main

Common Git & GitHub Commands

Check Status

git status

View Commit History

git log

View Short Log

git log --oneline

Pull Latest Changes

git pull origin main

Fetch Changes (Without Merge)

git fetch

Check Remote Repository

git remote -v

Create a New Branch

git branch new-branch

Switch Branch

git checkout new-branch

Create & Switch Branch (Shortcut)

git checkout -b new-branch

Merge Branch

git merge new-branch

Delete Branch

git branch -d new-branch

Undo & Recovery Commands

Undo Last Commit (Keep Changes)

git reset --soft HEAD~1

Undo Last Commit (Remove Changes)

git reset --hard HEAD~1

Revert a Commit (Safe for GitHub)

git revert <commit-id>

Stash Commands

Temporarily save changes:

git stash

View stash list:

git stash list

Apply stash:

git stash apply

Pull vs Fetch

  • git pull â†’ Fetch + Merge
  • git fetch â†’ Download changes only

Important Tips

  • Do not clone the same repo again
  • Use git pull to update files
  • Commit small, meaningful changes
  • Always check git status
